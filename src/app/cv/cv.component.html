Architecture

9

L'architecture logique

L'architecture logique repose sur une approche microservices avec des composants indépendants, une API Gateway comme point d'entrée unique, et Eureka pour la découverte dynamique. Chaque microservice dispose de sa propre base de données, assurant modularité, scalabilité et performance optimale.

Microservices API Gateway Eureka Base de données

L'architecture physique

L'architecture physique met en avant les interactions entre le client, le serveur front-end Angular, le back-end Spring Boot, et une base de données centralisée, assurant une communication fluide et une gestion sécurisée des données essentielles telles que les utilisateurs, missions, avis et CVs.

Angular Spring Boot Base de données Sécurité

L'architecture Microservice

L'architecture microservices divise l'application en services indépendants et spécialisés, favorisant modularité, flexibilité, et évolutivité. Chaque service gère une fonction spécifique, communique via des APIs, et peut être déployé ou mis à l'échelle de manière autonome.
